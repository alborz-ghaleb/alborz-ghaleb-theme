# ممیزی عملکرد Alborz Ghaleb 8.7.0

## دامنه ممیزی
ممیزی ایستا روی مسیر راه‌اندازی قالب، enqueue فایل‌ها، CSS/JS، تصاویر، کوئری‌ها، فونت‌ها، کش، دارک‌مود، صفحات چندزبانه، برگه‌ها، وبلاگ، آگهی‌ها و پنل کاربری انجام شد.

## بهینه‌سازی‌های اعمال‌شده

- چهار فایل CSS سراسری در `assets/css/core.css` تجمیع شدند؛ handleهای قبلی برای سازگاری حفظ شده‌اند.
- `dark-mode-fix.css` در `dark-mode.css` ادغام شد.
- `style.min.css` برای حالت Standalone ساخته شد؛ Child Theme همچنان از stylesheet خودش استفاده می‌کند.
- CSS توکار `page.php`، breadcrumb و آرشیو legacy به فایل خارجی قابل کش منتقل شد.
- CSS بلااستفاده `blog-archive.css` روی خانه وبلاگ، دسته‌ها و برچسب‌ها دیگر بارگذاری نمی‌شود.
- فایل غیرفعال `animations.js` دیگر دانلود نمی‌شود و عناصر reveal از اولین paint نمایان‌اند.
- باندل‌های اصلی JavaScript و CSSهای تولیدشده به‌صورت محافظه‌کارانه minify شدند.
- فقط اولین تصویر شاخص صفحه `eager/high` است؛ تصاویر مرتبط `lazy` باقی می‌مانند.
- `ORDER BY RAND()` از کوئری‌های مرتبط حذف شد.
- کوئری محبوب‌ترین نوشته‌ها یک ساعت cache می‌شود.
- `no_found_rows` و تنظیمات کم‌هزینه‌تر به کوئری‌های مرتبط افزوده شد.
- output buffering سراسری مربوط به پاک‌سازی Cloudflare به‌صورت پیش‌فرض غیرفعال شد.
- تعریف فونت self-hosted inline شد تا درخواست CSS بسیار کوچک فونت حذف شود؛ فایل‌های WOFF2 فقط در صورت استفاده دانلود می‌شوند.

## کنترل سلامت

- ۹۱ فایل PHP با parser بررسی شدند: بدون خطای نحوی.
- تمام فایل‌های CSS خروجی parse شدند: بدون خطای ساختاری.
- باندل‌های اصلی JavaScript با Node syntax-check شدند: بدون خطا.
- فایل ZIP نهایی با CRC کامل بررسی می‌شود.

## تنظیمات پیشنهادی سرور (خارج از محدوده قالب)

برای رسیدن به بیشترین سرعت واقعی، این موارد باید در هاست/CDN فعال شوند:

1. Full-page cache برای کاربران مهمان؛ صفحات ورود، داشبورد، ثبت آگهی و callback پرداخت مستثنا شوند.
2. Brotli یا Gzip برای HTML/CSS/JS/SVG/JSON.
3. Browser cache طولانی و `immutable` برای فایل‌های نسخه‌دار قالب.
4. PHP 8.2 یا جدیدتر همراه OPcache.
5. Redis Object Cache برای سایت پرترافیک.
6. Cloudflare APO یا CDN معادل، بدون Rocket Loader روی اسکریپت‌های پرداخت و پنل کاربری.
7. تبدیل تصاویر آپلودی بزرگ به WebP/AVIF و نگهداری ابعاد واقعی مورد نیاز.
8. پایگاه‌داده بهینه و پاک‌سازی دوره‌ای revision، transient منقضی و autoloadهای حجیم.

## مواردی که عمداً دست‌نخورده ماندند

- defer اجباری jQuery/Elementor فعال نشد؛ ممکن است فرم‌ها یا Elementor را خراب کند.
- async کردن CSS اختصاصی Elementor فعال نشد؛ خطر CLS و نمایش بدون استایل دارد.
- تأخیر اجباری اسکریپت‌های آمارگیر فعال نشد؛ نیازمند بررسی سرویس‌های واقعی سایت است.
- استایل بلوک‌های وردپرس حذف نشد؛ ممکن است محتوای Gutenberg را خراب کند.
- شمارنده بازدید و منطق پرداخت/ثبت آگهی تغییر معماری داده نشدند تا رفتار سایت حفظ شود.
